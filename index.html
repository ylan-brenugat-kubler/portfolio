Livrables (Code Source)

Le script permet de choisir un Pokémon par ID.

pokefiche.py 
import requests
import markdown
import sys

# Traduction des noms des statistiques
STATS_TRADUCTION = {
    "hp": "PV",
    "attack": "Attaque",
    "defense": "Défense",
    "special-attack": "Attaque spéciale",
    "special-defense": "Défense spéciale",
    "speed": "Vitesse"
}

def download_poke(id: int) -> dict:
    """Télécharge les données d'un Pokémon via l'API REST."""
    url = f"https://pokeapi.co/api/v2/pokemon/{id}"
    response = requests.get(url)
    if response.status_code == 200:
        return response.json()
    raise ValueError(f"Erreur : Pokémon '{id}' introuvable.")

def get_stats_for_pokemon(pokemon_name: str) -> dict:
    """Récupère les statistiques de base d'un Pokémon."""
    url = f"https://pokeapi.co/api/v2/pokemon/{pokemon_name}"
    response = requests.get(url)
    if response.status_code == 200:
        data = response.json()
        stats = {stat['stat']['name']: stat['base_stat'] for stat in data['stats']}
        return stats, data['sprites']['front_default']
    raise ValueError(f"Erreur : Pokémon '{pokemon_name}' introuvable.")

def poke_to_md(data: dict, filename: str) -> None:
    """Génère un fichier Markdown structuré."""
    name = data['name'].capitalize()
    height = data['height'] / 10
    weight = data['weight'] / 10
    types = ", ".join([t['type']['name'].capitalize() for t in data['types']])
    sprite_url = data['sprites']['front_default']

    markdown_content = f"""# Fiche Pokémon : {name}

![Image de {name}]({{sprite_url}})

- **Taille** : {height} m
- **Poids** : {weight} kg
- **Types** : {types}

"""
    stats, _ = get_stats_for_pokemon(name)
    markdown_content += "## Statistiques\n\n"
    for stat, value in stats.items():
        stat_name = STATS_TRADUCTION.get(stat, stat).capitalize()
        markdown_content += f"- **{{stat_name}}** : {{value}}\n"

    with open(filename, "w", encoding="utf-8") as file:
        file.write(markdown_content)

def convert(md_file: str, html_file: str) -> None:
    """Convertit un fichier Markdown en fichier HTML."""
    try:
        with open(md_file, 'r', encoding='utf-8') as md:
            html_content = markdown.markdown(md.read())
        with open(html_file, 'w', encoding='utf-8') as html:
            html.write(html_content)
    except Exception as e:
        print(f"Erreur : {e}")

if __name__ == "__main__":
    if len(sys.argv) > 1:
        pokemon_id = int(sys.argv[1])
        # Appel logique principale
    else:
        print("Usage : python3 pokefiche.py ")

SAÉ3Cyber03 : Réseau informatique sécurisé multi-sites

Compétence(s) : Cybersécurité et infrastructure multi-sites

Contexte et objectifs

Le projet consistait à répondre au cahier des charges de la société en expansion Pellet-SA (énergies renouvelables) nécessitant une infrastructure redondante, scalable et sécurisée. Le déploiement s'est déroulé en 3 phases progressives : la mise en place initiale de l'infrastructure (Phase 1), l'exploitation des vulnérabilités via des tests de pénétration (Phase 2), et enfin la remédiation et mise en œuvre des correctifs de sécurité (Phase 3).

Mon rôle

J'ai été impliqué sur de multiples aspects du déploiement :

  • Configuration d'équipements Cisco (commutateurs L2/L3 en stack) avec mise en œuvre de la redondance via un standard d'agrégation de liens et HSRP/VRRP en mode actif/passif.
  • Création de sous-réseaux et VLANs spécifiques selon l'organigramme (Services, Comptabilité, Terrain, Logistique, Administration) avec optimisation MSTP sur deux instances.
  • Déploiement sous hyperviseur Proxmox de deux machines virtuelles : une Debian fournissant les services DNS cache (Bind9) et TFTP, et une Windows Server 2019 Core hébergeant un annuaire ADDS (GX-Pellet.com.test) et un partage de fichiers SMB via la méthode ADGLP.
  • Sécurisation des accès aux équipements réseau par SSH.

Livrables (Extraits de Configuration)

SAÉ22 : Mesurer et caractériser un signal (Chasse au spectre)

Compétence(s) : Mesurer et caractériser un signal ou un système de télécommunication

Contexte et objectifs

Ce projet en binôme/trinôme consistait à réaliser une "chasse au spectre" de la téléphonie mobile sur le Campus 3 (Caen Sud). L'objectif était de relever le spectre RF (2G, 3G, 4G, 5G) à l'aide d'un récepteur RTL-SDR et de Matlab, en ciblant les émetteurs répertoriés sur cartoradio.fr.

Mon rôle

Dans le cadre de cette étude, nous nous sommes concentrés sur l'opérateur Orange :

  • Localisation de l'émetteur (17 Rue de Navarre, Cormelles-le-Royal).
  • Calcul théorique de la longueur optimale d'une antenne dipôle (15 cm pour 1 GHz).
  • Adaptation du script Matlab pour compenser le gain du RTL-SDR (retrait manuel des 40 dB).
  • Mesures de la bande de fréquences (400 MHz - 1700 MHz), identification de l'OBW (Occupied Bandwidth) du canal downstream (~10 MHz centré sur 813 MHz) et vérification du signal uplink (pic à 724 MHz en 5G).

Livrables (Rapport de mesures)

Note de synthèse et extraits des mesures spectrales sous Matlab :

# Calcul de la longueur optimale de l'antenne
Fréquence : f = 1 GHz = 1x10^9 Hz
Vitesse de la lumière : c = 3x10^8 m/s
Longueur d'onde : λ = c/f = 0.3 m = 30 cm
Longueur optimale d'une antenne dipôle : λ/2 = 15 cm

# Résultats d'analyse spectrale (RTL-SDR + Matlab)
- Plage globale : Sweep de 400 MHz à 1700 MHz.
- Zoom sur la bande d'Orange (811-821 MHz) :
  L'OBW (Occupied Bandwidth) à 99% de la puissance totale mesurée est d'environ 10 MHz, centrée autour de 813 MHz.
- Vérification de l'Uplink (émission du téléphone vers la station) : 
  La bande d'uplink de cet opérateur se situe entre 703 et 748 MHz. Un pic très marqué a été observé à 724 MHz en mode 5G, validant ainsi la détection de l'émission locale vers le réseau.
